Midterm Project

Purpose

This midterm project is designed to guide you to:

  1. identify appropriate methods for analyzing data.
  2. correctly fit models and summarize estimates.
  3. appropriately select between methods and assess the predictive ability of the final chosen method.
  4. apply the supervised-learning methods you have learned and practiced in the course thus far via a more free-form application.
  5. practice and receive feedback as preparation for the final project.

References

Material / Methods covered in:

  • Lessons 1 and 3-7 for supervised-learning methods.
  • Lessons 2 and 8 for model assessment and selection.

Tasks

Your project must analyze one of the datasets listed above and include the following:

Part 1

Select either the factor response or the quantitative response for the analysis in Part 2.

  1. Discuss why you selected the response variable (factor versus quantitative).
  2. Describe a possible practical purpose for the data analysis and an audience who might be interested in this purpose.

Part 2

Consider all other variables (excluding the two possible response variables) to be predictor variables. Then, for the response variable you selected:

  1. Identify two possible methods (from two different lessons) for predicting the response and provide justification for why these methods are appropriate for the data.
  2. Use cross-validation techniques to select between the two methods (and among any parameters needed for those methods).
  3. Add an outer level of cross-validation to further assess the predictive ability of the model selected.
  4. Fit the final selected model to the data and summarize and discuss the outcome of the fit. Include estimates of any model parameters.

Format

The project will consist of two pieces:

  1. A 1-2 page executive summary which reports your question, analysis, and results in a non-technical manner. Figures may be embedded with the text or included on an additional page. References, if any, may be included on an additional page. This should be in .doc, .docx, or .pdf format.
  2. A script containing the code you used to analyze the data. This should be a clean, commented, final version of the code, in either a .R file or a .Rmd file.

Assessment Checklist

Analysis (reflected in executive summary and code)

  • Employs two data mining techniques from this course that are appropriate to the question, each with full and correct application.
  • The analysis uses cross-validation appropriately for model selection.
  • The analysis uses double cross-validation appropriately for model assessment/validation.
  • Analysis is original; if resources other than ones provided for this course were used (including online discussions or posts such as Kaggle), these are cited in a References section at the end of the executive summary.
  • Final fitted model is stated (with estimated / selected parameters) and its use is described.

Executive summary

  • States a possible analysis purpose and explains why the target audience should be interested.
  • Discusses the types of variables and how they relate; this may involve appropriate figures for describing distributions and relationships.
  • Explains why the selected data mining techniques are appropriate. This may involve appropriate figures, such as plots for checking assumptions.
  • Writing is clear, professional, and in nontechnical language.
  • Includes at least 1 figure, which is easy to read and has a legend if appropriate.
  • Utility of the figure(s) to the analysis/results is clearly communicated.
  • Discussion of results states a specific recommendation for the target audience.

Computer code

  • Analysis code is well-commented. There is a clear connection between the code and the analyses and results described in the executive summary.
  • Analysis code corresponding to all aspects of executive summary (examination of data, application of methods, and model selection and assessment) is included.
  • Exploratory analyses unrelated to final results have been removed from the code.
  • Analysis code is able to be applied by a fresh user (your instructor in this case).
